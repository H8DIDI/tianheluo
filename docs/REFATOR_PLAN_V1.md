# 萤合落改造计划 V1

## Goal

把 `yingheluo-v3` 从“功能堆叠但体验割裂的烟花后台”重构成“手机优先、点击即放、支持编排、支持图案/几何烟花、可继续升级性能”的开源烟花产品。

## Phase 1: 爽玩模式优先

目标：一打开就能放烟花。

1. 主页面变成全屏舞台
2. 点击/触摸舞台即时发射
3. 手机端底部只保留核心入口
4. 编排、阵地、AI 退到第二层

## Phase 2: 视觉引擎升级

目标：烟花真正好看。

1. 继续拆 `FireworksScene`
2. 重构粒子分层：壳弹、核心、星点、余烬、拖尾
3. 引入更强 Bloom / 闪光 / 冲击感
4. 逐步从 CPU 粒子迁到更高效的实例或 shader 粒子

## Phase 3: 图案 / 文字 / 几何烟花

目标：建立差异化能力。

1. 先做预设图案：心形、圆环、星形
2. 再做文字输入转点阵
3. 再做几何体：sphere、torus、cube frame、helix
4. 为未来立体点云烟花留接口

## Phase 4: 编排与音乐同步

目标：从爽玩升级到演出工具。

1. 用统一 cue 主链路收敛播放系统
2. 时间轴加入 ruler、markers、regions
3. 引入波形与音乐同步
4. 分离 runtime scheduler 与 editor timeline

## Phase 5: 阵地编辑器重构

目标：把地图/阵地从简单编辑变成专业工具。

1. 加入框选、手柄、吸附、对齐
2. 建立 undo/redo
3. 统一编辑模式状态机
4. 将阵地与图案布局能力逐步融合

## Phase 6: 管理与资源系统收敛

目标：只保留真正有用的管理面，不再喧宾夺主。

1. 精简 admin / manager 冗余页面
2. 重做烟花库、模板库、音频库
3. 用命令面板替代部分复杂入口
4. 资源管理集中化

## 立即执行顺序

1. 下载并研究高优先参考仓库源码
2. 先改主舞台与点击即放
3. 再改粒子视觉与 Bloom
4. 再开始图案/几何烟花基础设施
